## 오늘의 문제 - 프로그래머스 Lv.2 [<a href="https://school.programmers.co.kr/learn/courses/30/lessons/118667" target="_blank" className="a-mdx">두 큐 합 같게 만들기</a>]

### 문제 설명

길이가 같은 두 개의 큐가 주어집니다. 하나의 큐를 골라 원소를 추출(pop)하고, 추출된 원소를 다른 큐에 집어넣는(insert) 작업을 통해 각 큐의 원소 합이 같도록 만들려고 합니다. 이때 필요한 작업의 최소 횟수를 구하고자 합니다. 한 번의 pop과 한 번의 insert를 합쳐서 작업을 1회 수행한 것으로 간주합니다.

큐는 먼저 집어넣은 원소가 먼저 나오는 구조입니다. 이 문제에서는 큐를 배열로 표현하며, 원소가 배열 앞쪽에 있을수록 먼저 집어넣은 원소임을 의미합니다. 즉, pop을 하면 배열의 첫 번째 원소가 추출되며, insert를 하면 배열의 끝에 원소가 추가됩니다. 예를 들어 큐 **[1, 2, 3, 4]**가 주어졌을 때, pop을 하면 맨 앞에 있는 원소 1이 추출되어 **[2, 3, 4]**가 되며, 이어서 5를 insert하면 **[2, 3, 4, 5]**가 됩니다.

다음은 두 큐를 나타내는 예시입니다.

<div className={"dark:bg-secondary-dark p-1 pl-3 rounded-lg"}>
  queue1 = [3, 2, 7, 2]
  queue2 = [4, 6, 5, 1]
</div>

두 큐에 담긴 모든 원소의 합은 30입니다. 따라서, 각 큐의 합을 15로 만들어야 합니다. 예를 들어, 다음과 같이 2가지 방법이 있습니다.

1. queue2의 4, 6, 5를 순서대로 추출하여 queue1에 추가한 뒤, queue1의 3, 2, 7, 2를 순서대로 추출하여 queue2에 추가합니다. 그 결과 queue1은 [4, 6, 5], queue2는 [1, 3, 2, 7, 2]가 되며, 각 큐의 원소 합은 15로 같습니다. 이 방법은 작업을 7번 수행합니다.
2. queue1에서 3을 추출하여 queue2에 추가합니다. 그리고 queue2에서 4를 추출하여 queue1에 추가합니다. 그 결과 queue1은 [2, 7, 2, 4], queue2는 [6, 5, 1, 3]가 되며, 각 큐의 원소 합은 15로 같습니다. 이 방법은 작업을 2번만 수행하며, 이보다 적은 횟수로 목표를 달성할 수 없습니다.
따라서 각 큐의 원소 합을 같게 만들기 위해 필요한 작업의 최소 횟수는 2입니다.

길이가 같은 두 개의 큐를 나타내는 정수 배열 **queue1**, **queue2**가 매개변수로 주어집니다. 각 큐의 원소 합을 같게 만들기 위해 필요한 작업의 최소 횟수를 return 하도록 solution 함수를 완성해주세요. 단, 어떤 방법으로도 각 큐의 원소 합을 같게 만들 수 없는 경우, -1을 return 해주세요.

### 제한 사항

- 1 ≤ **queue1**의 길이 = **queue2**의 길이 ≤ 300,000
- 1 ≤ **queue1**의 원소, **queue2**의 원소 ≤ 109
- 주의: 언어에 따라 합 계산 과정 중 산술 오버플로우 발생 가능성이 있으므로 long type 고려가 필요합니다.

### 입출력 예

| queue1       | queue2        | result |
|--------------|---------------|--------|
| [3, 2, 7, 2] | [4, 6, 5, 1]  | 2      |
| [1, 2, 1, 2] | [1, 10, 1, 2] | 7      |
| [1, 1]       | [1, 5]        | -1     |

### 입출력 예 설명

입출력 예 #1

문제 예시와 같습니다.

입출력 예 #2

두 큐에 담긴 모든 원소의 합은 20입니다. 따라서, 각 큐의 합을 10으로 만들어야 합니다. queue2에서 1, 10을 순서대로 추출하여 queue1에 추가하고, queue1에서 1, 2, 1, 2와 1(queue2으로부터 받은 원소)을 순서대로 추출하여 queue2에 추가합니다. 그 결과 queue1은 [10], queue2는 [1, 2, 1, 2, 1, 2, 1]가 되며, 각 큐의 원소 합은 10으로 같습니다. 이때 작업 횟수는 7회이며, 이보다 적은 횟수로 목표를 달성하는 방법은 없습니다. 따라서 7를 return 합니다.

입출력 예 #3

어떤 방법을 쓰더라도 각 큐의 원소 합을 같게 만들 수 없습니다. 따라서 -1을 return 합니다.

## 풀이 과정

### 그리디

먼저 각 배열을 순회하면서 queue로 바꾸어주고 합도 구한다. 문제의 힌트대로 수의 크기가 int범위를 넘어설 수 있으므로 long 자료형을 사용해주었다. 두 배열의 합을 더하면 모든 원소의 합을 구할 수 있다. 이 합이 홀수이면 절대 합이 같게 만들 수 없으므로 -1을 return한다.

여기서 두 배열의 합을 같게 하려면
- queue1의 합이 전체 합의 절반 초과이면 queue1에서 pop을 하고 queue2에 추가한다.
- queue1의 합이 전체 합의 절반 미만이면 queue2에서 pop을 하고 queue1에 추가한다.

다음과 같은 로직으로 수행한다. 원소의 합을 구할 때는 기존에 구했던 합에서 더하고 빼는 작업으로 최적화 할 수 있다.

작업의 최대 횟수는 [1, 1, 1, 1], [1, 1, 7, 1] 등 (전체 원소 수) + 1이다. 그래서 최대 반복 횟수가 (전체 원소 수) + 3 이다. 이 작업 횟수를 초과하면 목표 달성이 불가능한 것이므로 -1을 return한다.

#### 정답

```java
import java.util.*;

class Solution {
    public int solution(int[] queue1, int[] queue2) {
        long sum1 = 0;
        long sum2 = 0;
        Deque<Integer> q1 = new LinkedList<>();
        Deque<Integer> q2 = new LinkedList<>();
        for (int i = 0; i < queue1.length; i++) {
            sum1 += (long) queue1[i];
            sum2 += (long) queue2[i];
            q1.offer(queue1[i]);
            q2.offer(queue2[i]);
        }

        long sum = sum1 + sum2;
        if (sum % 2 == 1) return -1; // 합이 홀수이면 절대 목표 달성이 불가능
        sum /= 2;
        int answer = 0;
        for (int i = 0; i < queue1.length * 2 + 3; i++) {
            if (sum1 > sum) {
                int poll = q1.poll();
                q2.offer(poll);
                sum1 -= poll;
                sum2 += poll;
            } else if (sum1 < sum) {
                int poll = q2.poll();
                q1.offer(poll);
                sum1 += poll;
                sum2 -= poll;
            } else {
                return answer;
            }
            answer++;
        }
        return -1;
    }
}
```

## 정리

사실 작업의 최대 횟수를 정하는 부분에서 그 값을 (전체 원소 수) + 2라고 생각했다. 기본적으로 상술한 테스트 케이스처럼 최대 (전체 원소 수) * 2 + 1 의 작업 횟수가 필요하고 반복문 내 같은 부분을 확인하는 로직으로 인해 한번 더 반복문을 실행해야 한다. 하지만 1번 테스트 케이스를 통과하지 못했다. 그래서 반복 횟수를 1 더해봤는데 통과가 되었다. 아무리 생각해 보아도 왜 반복 횟수가 1회 더 필요한지는 잘 모르겠다.
