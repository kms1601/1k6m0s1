### 문제 설명

**[본 문제는 정확성과 효율성 테스트 각각 점수가 있는 문제입니다.]**

당신은 코딩 테스트를 준비하기 위해 공부하려고 합니다. 코딩 테스트 문제를 풀기 위해서는 알고리즘에 대한 지식과 코드를 구현하는 능력이 필요합니다.

알고리즘에 대한 지식은 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span>, 코드를 구현하는 능력은 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>코딩력</span>이라고 표현합니다. <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span>과 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>코딩력</span>은 0 이상의 정수로 표현됩니다.

문제를 풀기 위해서는 문제가 요구하는 일정 이상의 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span>과 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>코딩력</span>이 필요합니다.

예를 들어, 당신의 현재 <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span>이 15, <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>코딩력</span>이 10이라고 가정해보겠습니다.

<span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span>을 높이기 위해 알고리즘 공부를 합니다. <span className={"bg-secondary dark:bg-secondary-dark p-1 rounded"}>알고력</span> 1을 높이기 위해서 1의 시간이 필요합니다.

## 풀이 과정

### 목표 구하기

먼저 도달해야 할 알고력과 코딩력을 알아야한다. 모든 문제를 풀 수 있어야 하므로 문제 중 가장 높은 알고력과 코딩력이 도달해야할 수치이다.

### DP 배열

2차원 배열로 DP 배열을 만든다. 이 배열의 각 칸의 의미는 dp[a][c] - 알고력 a, 코딩력 c에 도달하기 위한 최단 시간이다.

이 배열의 크기는

- 행 - (목표 알고력, 초기 알고력 중 최댓값) + 1
- 열 - (목표 코딩력, 초기 코딩력 중 최댓값) + 1

와 같이 설정한다. 목표값만이 아닌 초기값도 고려하는 이유는 초기값 위치의 값을 0으로 초기화할 것이기 때문이다.

그리고 배열을 전부 INF값으로 채우고 초기 알고력, 코딩력에 해당하는 위치의 값을 0으로 초기화한다(그 수치에 도달하기 위헤 시간이 필요하지 않음).

### DP 수행

위에서 만든 DP 배열을 전부 순회한다.

- 만약 값이 INF라면 해당 수치로 만들 수 있는 경우가 없는 것이므로 넘어간다.
- 만약 값이 INF가 아니라면 모든 문제를 순회한다.
    - 각 문제의 요구 알고력, 코딩력을 현재 알고력, 코딩력이 넘지 못했다면 넘어간다(풀 수 없음).
    - 그렇지 않다면 현재 알고력, 코딩력에 문제의 증가값을 더한다. 만약 이 값이 배열의 범위를 넘어선다면 목표값으로 제한한다.
    - 이렇게 얻은 새로운 알고력, 코딩력에 해당하는 배열값의 최솟값을 갱신한다.
    - 마지막으로 1의 시간을 써서 각 수치를 올리는 경우를 처리한다.

이렇게 DP 배열을 전부 채우고 배열의 오른쪽 아래 값을 반환한다.

### 정답

```java
import java.util.*;

class Solution {
    private final int INF = 1_000_000_000;

    public int solution(int alp, int cop, int[][] problems) {
        // 목표값 찾기
        int trtAlp = alp, trtCop = cop;
        for (int[] problem : problems) {
            trtAlp = Math.max(trtAlp, problem[0]);
            trtCop = Math.max(trtCop, problem[1]);
        }

        int[][] dp = new int[trtAlp + 1][trtCop + 1];

        // 각 배열의 값을 INF로 채우고 초기 수치에 해당하는 배열값을 0으로 한다.
        for (int i = 0; i <= trtAlp; i++) {
            Arrays.fill(dp[i], INF);
        }
        dp[alp][cop] = 0;

        for (int a = 0; a <= trtAlp; a++) {
            for (int c = 0; c <= trtCop; c++) {
                if (dp[a][c] == INF) continue; // 해당 수치로 도달할 수 없으면 넘어간다.

                for (int[] problem : problems) {
                    if (a < problem[0] || c < problem[1]) continue; // 문제를 풀 수 없다면 넘어간다.

                    // 배열의 범위를 넘지 않게 값을 제한한다.
                    int na = Math.min(trtAlp, a + problem[2]);
                    int nc = Math.min(trtCop, c + problem[3]);
                    dp[na][nc] = Math.min(costs[na][nc], dp[a][c] + problem[4]); // 최솟값 갱신
                }

                // 각 수치를 1 올리는 경우를 처리
                if (a != trtAlp) dp[a + 1][c] = Math.min(dp[a + 1][c], dp[a][c] + 1);
                if (c != trtCop) dp[a][c + 1] = Math.min(dp[a][c + 1], dp[a][c] + 1);
            }
        }

        return dp[trtAlp][trtCop];
    }
}
```
